Team! User Manuals Builder.
A company that is planning to release a new hardware product or update an existing one, 
needs to collect data and structure user manuals. The data of user manuals should include fields like 
product name, table of content, product overview, instructions for each feature and danger warnings.

Capabilities
- Different people can access the manuals, can edit their content
- We need manual versioning, so we can roll back the manual data back
- A list of Danger warnings that I can select of for this manual (reducing manual work)
- Multi-language support for the Manuals, so manuals can be created in different languages.
- Automated language translations
- Automated parsing of a PDF manual.
- Preview of the PDF files
- PDF exports
- Auth and user management. As an admin I can assing a Manual to a specific User, so they can work with it.
- Toast notifications about changes coming from other users.
- Favorite manuals for the user. Users can select favorite and filter them on the list of manuals.
- Web and Mobile web versions are out priorities. Design must be stunning and responsive.

Out of scope 
- email notification  
- layout management/customizations is out of scope
- picture storage and including pictures into the layout of the manuals
- apps like ios/android
